# 데이터 갱신 런북 (분기별)

병원 데이터는 **건강보험심사평가원(HIRA) 공공데이터**의 고정 스냅샷입니다.
HIRA는 **분기별**로 갱신하므로, 새 분기가 나오면 아래 6단계로 재빌드합니다. (자동화하지 않음 — 사유는 맨 아래)

> 실시간 휴·폐업/진료시간은 데이터에 없으며, 각 병원 카드의 **"최신 확인"** 링크(네이버 검색·지도·뉴스·홈페이지)로 사용자가 직접 확인합니다.

---

## 갱신 절차

1. **다운로드** — HIRA 보건의료빅데이터개방시스템 / data.go.kr 에서 최신 분기 엑셀 **2개**:
   - `병원정보서비스` (기본정보 + 좌표)
   - `의료기관별상세정보서비스 — 진료과목정보`

2. **배치** — `build/raw/` 에 아래 파일명 규칙대로 저장 (괄호 안 분기를 새 분기로):
   ```
   build/raw/1.병원정보서비스(2026.6.).xlsx
   build/raw/5.의료기관별상세정보서비스_03_진료과목정보(2026.6.).xlsx
   ```

3. **상수 수정** — `build/build_data.py` 상단에서 분기만 교체:
   - `DATA_VER = "2026.6"`
   - `SRC_LABEL = "건강보험심사평가원 · 전국 병의원 및 약국 현황(2026.6.)"`
   - `HOSP` / `DEPT` 파일명의 분기 (2번 파일명과 일치)
   - `generated` / `region` 은 **자동 계산**되므로 손대지 않음

4. **빌드**
   ```powershell
   python build/build_data.py
   ```

5. **검증 (눈으로 확인)** — 스크립트 출력에서:
   - `→ 대상 NN,NNN곳` 이 **7만 이상**인지 (전국 기준 ~79,000)
   - 콘솔 마지막의 종별 분포·심층 후보 목록이 정상인지
   - ⚠ 값이 급감했으면 **HIRA가 컬럼 순서를 바꾼 것** — `build_data.py:26-33`의 컬럼 인덱스를 새 파일에 맞춰 점검

6. **캐시 무효화 + 배포** — `index.html` 의 데이터 캐시 버전을 올린 뒤 커밋·푸시:
   ```
   <script src="assets/data.js?v=n7"></script>   →  ?v=n8 로 증가
   ```
   ```powershell
   git add assets/data.js index.html build/
   git commit -m "chore: HIRA 2026.6 분기 데이터 갱신"
   git push        # GitHub Pages 자동 반영
   ```

---

## 갱신 시 자동 반영되는 것

- **데이터 기준일** — `META.version` → 푸터 "데이터 기준 …" 라벨
- **빌드일** — `META.generated` (= 빌드한 날짜, `date.today()`)
- **수록 범위** — `META.region` → 마스트헤드 "대상 지역" (시도 수 자동 집계)

## 수록 범위 바꾸기 (선택)

`build_data.py` 상단 `TARGET_SIDO` / `TARGET_GU` 로 한정 가능:
- `TARGET_SIDO=None` → 전국 (기본)
- `TARGET_SIDO="서울"` → 서울 전체
- `TARGET_GU={"강남구","서초구"}` → 특정 자치구만

라벨(`region`)은 설정에 맞춰 자동 생성됩니다.

---

## 왜 자동화하지 않나

- HIRA는 **안정적인 bulk API가 없음** — 포털 첨부파일 다운로드라 스크립트가 잘 깨짐
- 분기마다 **컬럼 순서·이름이 바뀌는 경우**가 있어 사람이 한 번 눈으로 봐야 안전 (`build_data.py`는 컬럼을 인덱스로 읽음)
- 연 4회 작업이라 자동화 ROI가 낮음

따라서 **수동 + 이 런북 + 5단계 검증**이 가장 안전하고 저비용입니다.
